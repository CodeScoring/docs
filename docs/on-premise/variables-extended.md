# Описание переменных
Данный раздел содержит описание переменных, используемых при установке платформы, но не описанных в разделах [Установка системы в Docker](/on-premise/installation) и [Работа системы в Kubernetes](/on-premise/installation-in-k8s)

**Важно!** Перед внесением любых изменений в стандартное значение переменных необязательных к заполнению во время установки, проконсультируйтесь с технической поддержкой.

## **env.template**
По умолчанию интеграция с Sentry отключена `SENTRY_ENABLE=False`, при необходимости использования Sentry, для отправки данных об ошибках и других событий системы укажите `SENTRY_ENABLED=True` и заполните следующие переменные:

- **SENTRY_DNS** - идентификатор проекта внутри Sentry в который будут отправляться данные;
- **SENTRY_ENVIRONMENT** - Окружение к которому будут относится отправляемые данные;
- **SENTRY_RELEASE** - Релиз с которым будут связаны отправляемые данные `app.env`.

## **app.env**

- Переменная `PATH` определяет путь исполняемых файлов, файлов виртуального окружения (venv) и стандартные системные пути;
     ```bash
     PATH=/venv/bin:${PATH}:/jscpd/node_modules/jscpd/bin:/sbin:/bin:/usr/bin:/usr/sbin:/usr/local/bin:/usr/local/sbin:/root/bin
     ```

- Переменные, указывающие на расположение основных файлов инсталляции:
    - **HOME** =/ipcs-backend;
    - **BASE_DIR** =/ipcs-backend;
    - **AZURE_DEVOPS_CACHE_DIR** =/ipcs-backend.

- **JOHNNY_BIN** =/agents/johnny-linux-amd64 - путь к исполняемому файлу агента сканирования JOHNY;

- **HASHER_BIN** =/agents/hasher-linux-amd64 - Путь к исполняемому файлу агента `HASHER`, отвечающего за хеширование файлов;

- **INDEX_API_URL** =https://index.codescoring.ru - URL-адрес API для индекса, необходимого для обновления базы знаний. Данный параметр не требует изменений;

- **DJANGO_CACHES_REDIS_URL** =redis://redis:6379/1 - адрес сервера Redis, используемому в качестве системы кэширования Django;

- **ANALYSIS_ROOT** =/analysis-root - корневая директория для клонирования репозиториев для анализа. Файлы копируются как временные и после анализа удаляются (используется только vcs проектами);

- **MEDIA_ROOT** =/media-root - рабочая директория для промежуточного хранения скачиваемых и загружаемых файлов(таких как отчёт, SBOM и т.д);

- **NODE_PATH** =/jscpd/node_modules - директория расположения `node_modules`;

- **ALLOWED_HOSTS** =`*` - разрешенные хосты для Django (по умолчанию разрешены все хосты).

**HUEY** - сервис отвечающий за очереди и задачи, количество воркеров рекомендуется изменять, только после консультации со специалистами технической поддержки.

* **HUEY_REDIS_URL** =redis://redis:6379/0 - адрес для подключения к серверу Redis, используемому библиотекой Huey (очередь задач);
* **HUEY_WORKERS** =8 - количество воркеров HUEY разбирающих очереди tasks-main;
* **HUEY_HIGH_PRIORITY_WORKERS** =8 - количество воркеров HUEY для высокоприоритетных задач;
* **HUEY_OSA_PACKAGE_SCAN_WORKERS** =4 - количество воркеров HUEY для модуля сканирования пакетов OSA;
* **HUEY_OSA_CONTAINER_IMAGE_SCAN_WORKERS** =4 - количество воркеров HUEY для модуля сканирования контейнеров OSA;
* **HUEY_POLICY_WORKERS** =4 - количество воркеров HUEY для работы политик безопасности;
* **HUEY_TQI_WORKERS** =4 - количество воркеров HUEY для модуля TQI;
* **HUEY_SECRETS_WORKERS** =4 - количество воркеров HUEY для модуля поиска секретов;
* **HUEY_MEDIA_WORKERS** =4 - количество воркеров Huey для отвечающих за загрузку медиа;
* **HUEY_SCA_EXTERNAL_SCAN_WORKERS** =4 - количество воркеров отвечающих за очередь анализа, запущенного через агент Джонни `(tasks-sca-external-scan)`;
* **HUEY_WEBHOOKS_WORKERS** =2 - количество воркеров HUEY отвечающих за работу вебхуков;
* **HUEY_WORKER_MAX_TASKS** =500 - максимальное количество задач, которое может обработать один воркер HUEY.
